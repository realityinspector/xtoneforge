## When parallel orchestration beats collaborative coding

Claude Code Teams and Stoneforge represent two fundamentally different approaches to scaling AI-assisted development.

**Claude Code Teams** gives your team a shared, managed workspace where Claude agents collaborate on code. It's polished, integrated, and backed by Anthropic's infrastructure. You get a smooth experience without managing anything yourself.

**Stoneforge** takes a different approach: it's an orchestration layer that coordinates multiple independent AI agents working in parallel. Each agent gets its own isolated git worktree, works on its own task, and a Steward agent handles merging everything together.

### The multi-agent advantage

The core architectural difference matters most when you need to scale. With Stoneforge, you can dispatch 10 tasks and have 10 agents working simultaneously — each in complete isolation. There's no shared workspace, no stepping on each other's code, and no waiting for one agent to finish before another starts.

```bash
# Create a plan with parallel tasks
sf plan create --title "Sprint 22 Features"

# These tasks get dispatched to separate agents automatically
sf task create --title "Add OAuth2 integration" --plan "Sprint 22 Features"
sf task create --title "Refactor payment service" --plan "Sprint 22 Features"
sf task create --title "Build notification system" --plan "Sprint 22 Features"
sf task create --title "Add API rate limiting" --plan "Sprint 22 Features"
```

### Your code stays on your machine

Stoneforge runs entirely on your infrastructure. Your source code, prompts, and context never leave your machine. For teams in regulated industries, working with proprietary code, or simply wanting full control — this is non-negotiable.

### Use any agent you want

Stoneforge isn't locked to a single AI provider. Use Claude Code for complex architectural work, Codex for rapid prototyping, or OpenCode with local models for air-gapped environments. Mix and match based on what each task needs.
