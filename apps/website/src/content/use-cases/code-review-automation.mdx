Stoneforge's [Steward agents](https://docs.stoneforge.ai/core-concepts/agents/) automate the tedious parts of code review, letting your senior developers focus on what matters — architecture and design decisions.

### Consistent, thorough first pass

Every PR gets the same level of scrutiny. The Steward agent checks for style issues, common bugs, security vulnerabilities, and test coverage — catching the mechanical issues before a human reviewer ever opens the PR.

```bash
# The Steward agent reviews every merge request automatically.
# No configuration needed — it's built into the orchestration flow.

# Customize review standards in your workspace:
# .stoneforge/prompts/steward.md:
# "Review for: error handling completeness, input validation,
#  SQL injection risks, consistent naming conventions, test coverage
#  for new code paths. Flag any TODO comments without linked issues."
```

### Beyond linting

Linters catch syntax issues. Stoneforge's review agents understand *intent*. They can identify logic errors, suggest more efficient algorithms, flag race conditions, and spot missing edge cases — the kinds of issues that require understanding the full context of a change.

### Custom review checklists

Define what matters to your team. Create [custom Steward configurations](https://docs.stoneforge.ai/guides/custom-stewards/) that enforce your specific standards — from API design patterns to error handling requirements to accessibility compliance.

```bash
# Example: Custom Steward for API changes
# Checks: request validation, response format, error codes,
# rate limiting, authentication, pagination, versioning
```

### Actionable feedback, not noise

Review comments include specific fix suggestions, not vague observations. Instead of "this could be improved," the Steward says "this function doesn't handle the null case on line 42 — add a guard clause like `if (!user) return null;`".

### Speed up the review cycle

With mechanical issues caught automatically, human reviewers can approve PRs faster. Teams using Stoneforge for automated review report 40-60% reduction in review cycle time and significantly fewer post-merge bugs.

```bash
# Review flow:
# 1. Agent completes task → MR created
# 2. Steward reviews automatically → comments posted
# 3. Agent fixes Steward feedback → MR updated
# 4. Human reviewer sees clean PR → fast approval
# 5. Steward merges → done
```
